# event-security 本地部署指南

## 1. 环境要求

| 组件 | 版本 | 用途 |
|------|------|------|
| Java | JDK 8 | Flink 引擎 |
| Node.js | v24+ | 后端服务 |
| Docker | 24+ | 容器化中间件 |
| Git | 2.x | 代码管理 |

## 2. 中间件部署

项目依赖 5 个中间件，全部通过 Docker 或系统服务运行。

### 2.1 PostgreSQL（Docker）

```bash
docker run -d --name nsm-postgres \
  -e POSTGRES_USER=soc \
  -e POSTGRES_PASSWORD=123456 \
  -e POSTGRES_DB=psoc001db \
  -p 5432:5432 \
  postgres:14-alpine
```

**说明**：应用首次启动时自动创建以下表并填充种子数据：

- `ice_rule` — 安全事件模型规则（由 nsm-event-engine 写入）
- `attack_scene` — 攻击场景分类（自动填充 13 条）
- `ice_analysis_task` — 分析任务
- `audit_log` — 操作审计日志

### 2.2 Elasticsearch 7.16.3（系统安装）

```bash
# 下载（如未安装）
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-7.16.3-linux-x86_64.tar.gz
tar -xzf elasticsearch-7.16.3-linux-x86_64.tar.gz

# 配置单节点集群
echo "cluster.name: event-security" >> elasticsearch-7.16.3/config/elasticsearch.yml
echo "node.name: node-1" >> elasticsearch-7.16.3/config/elasticsearch.yml

# 启动（不能用 root，需创建专用用户）
useradd -m esuser
chown -R esuser:esuser elasticsearch-7.16.3/
su - esuser -c "./elasticsearch-7.16.3/bin/elasticsearch -d"
```

**验证**：`curl http://localhost:9200/`

应用启动时自动创建 4 个索引：

| 索引 | 用途 |
|------|------|
| `incident-index` | 安全事件（聚合后） |
| `alert-index` | 告警记录 |
| `log-index` | 原始日志 |
| `entity-index` | 攻击实体（attacker/victim） |

### 2.3 Redis（Docker）

```bash
# 单节点
docker run -d --name redis1 \
  -p 7001:6379 \
  redis:7-alpine

# 如需集群（可选）
docker run -d --name redis2 -p 7002:6379 redis:7-alpine
docker run -d --name redis3 -p 7003:6379 redis:7-alpine
```

**用途**：白名单缓存、聚合状态、图谱缓存。

### 2.4 Kafka（Docker）

```bash
docker run -d --name nsm-kafka \
  -p 9092:9092 \
  -e KAFKA_NODE_ID=1 \
  -e KAFKA_PROCESS_ROLES=broker,controller \
  -e KAFKA_LISTENERS='PLAINTEXT://0.0.0.0:9092,CONTROLLER://0.0.0.0:9093' \
  -e KAFKA_ADVERTISED_LISTENERS='PLAINTEXT://localhost:9092' \
  -e KAFKA_CONTROLLER_LISTENER_NAMES=CONTROLLER \
  -e KAFKA_LISTENER_SECURITY_PROTOCOL_MAP=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT \
  -e KAFKA_CONTROLLER_QUORUM_VOTERS='1@localhost:9093' \
  -e KAFKA_INTER_BROKER_LISTENER_NAME=PLAINTEXT \
  -e CLUSTER_ID='MkU3OEVBNTcwNTJENDM2Qk' \
  confluentinc/cp-kafka:7.6.0
```

**说明**：KRaft 模式，无需 Zookeeper。应用启动时自动创建以下 Topic：

| Topic | 生产者 | 消费者 |
|-------|--------|--------|
| `nsm-xdr-events` | Flink | Node.js Aggregator |
| `nsm-original-alert` | Flink | Node.js Searcher |
| `nsm-aggregated-alert` | Flink Aggregator | Node.js Searcher |
| `nsm-incident-task` | Node.js | — |
| `nsm-incident-events` | — | — |

## 3. 项目部署

### 3.1 克隆代码

```bash
cd ~/app
git clone https://github.com/windDcl/event-security.git
cd event-security
```

### 3.2 安装依赖

```bash
npm install
```

依赖列表（极简）：

- `pg` — PostgreSQL 客户端
- `@elastic/elasticsearch` — ES 7.x 客户端
- `kafkajs` — Kafka 生产/消费
- `ioredis` — Redis 客户端
- `yaml` — DSL 解析

### 3.3 启动服务

```bash
# 开发模式
npm run dev

# 或直接
node server/index.js
```

服务监听：`http://127.0.0.1:4173`

**启动过程**（自动完成）：

1. 初始化 PostgreSQL 表结构 + 种子数据
2. 创建 ES 索引（incident/alert/log/entity）
3. 创建 Kafka Topic（如不存在）
4. 启动 Kafka Consumer（监听 `nsm-aggregated-alert`）
5. 启动 HTTP Server（API + 静态前端）

### 3.4 访问前端

浏览器打开 `http://127.0.0.1:4173`

## 4. Flink 引擎部署（数据源）

Flink 引擎是事件数据的生产者，需要单独部署。

### 4.1 克隆引擎代码

```bash
cd ~/app
git clone https://github.com/windDcl/nsm-event-engine.git
cd nsm-event-engine
```

### 4.2 编译打包

```bash
mvn clean package -DskipTests
```

### 4.3 启动规则匹配引擎

```bash
java -cp target/nsm-event-engine-1.0-SNAPSHOT.jar \
  com.pbc.nsm.engine.EngineMainEntry
```

**职责**：读取 PG 中的 `ice_rule` 规则，匹配原始事件，输出告警到 Kafka。

### 4.4 启动聚合引擎

```bash
java -cp target/nsm-event-engine-1.0-SNAPSHOT.jar \
  com.pbc.nsm.engine.EngineAggregatorEntry
```

**职责**：消费告警，按 victim 维度聚合，写入 ES `incident-index` 和 `alert-index`，输出聚合告警到 `nsm-aggregated-alert`。

## 5. 端到端数据流

```
外部事件源
  ↓ JSON
Flink EngineMainEntry（规则匹配）
  ↓ 原始告警 → Kafka: nsm-original-alert
Flink AggregatorFunction（聚合）
  ↓ incident → ES: incident-index
  ↓ alert    → ES: alert-index
  ↓ 聚合告警 → Kafka: nsm-aggregated-alert
Node.js Searcher（DSL 执行）
  ↓ 图谱数据 → 前端展示
  ↓ evidence/impact → ES 查询
Node.js Server（HTTP API）
  ↓ REST API
前端 SPA（public/index.html）
```

## 6. 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `4173` | HTTP 服务端口 |

按顺序启动后，依次检查：

```bash

# 2. Elasticsearch

# 期望：4 个索引（incident/alert/log/entity）

```

## 8. 常见问题

### Kafka Docker 镜像选择

使用 `confluentinc/cp-kafka:7.6.0`，**不要**用 `apache/kafka:3.7.0`（已知 producer 成功但 consumer 读不到消息的 bug）。

### Flink 消费不到新消息

检查 consumer group offset：
```bash
docker exec nsm-kafka kafka-consumer-groups \
  --bootstrap-server localhost:9092 \
  --describe --group nsm-aggregator-test
```

如 offset 落后，重置：
```bash
docker exec nsm-kafka kafka-consumer-groups \
  --bootstrap-server localhost:9092 \
  --group nsm-aggregator-test \
  --reset-offsets --to-earliest --topic nsm-original-alert --execute
```

### ES 索引映射冲突

删除索引重建（会丢失数据）：
```bash
curl -X DELETE http://localhost:9200/alert-index
# 重启 Node.js 服务自动重建
```

### PG 密码/端口不对

确认 Docker 容器：
```bash
docker ps | grep postgres
# 检查端口映射和环境变量
```
