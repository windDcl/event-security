## 2026-06-18 第四阶段：Flink 聚合引擎

### 已完成

- M17 自动聚合引擎迁移到 Flink
  - `AggregatorFunction.java` — Flink KeyedProcessFunction，核心聚合逻辑
  - `EngineAggregatorEntry.java` — 独立 Flink Job 入口
  - `FlinkUtil.java` — 新增 `getKafkaSource(earliest)` 重载
  - 告警标准化：`src_ip→attacker`, `dst_ip→victim`, `alarm_name→title` 等
  - 模型匹配：从 PG `ice_rule.raw` 加载 entryAlerts，按 `rule_id` 匹配
  - victim 维度聚合：Flink KeyedState 维护 incidentId + alertCount
  - ES 直写：HttpURLConnection 写入 incident-index + alert-index
  - 双 PG 数据源：nsm 库 + psoc001db 库（ice_rule 表）
  - ISO 8601 日期格式兼容 ES

- Searcher 适配
  - 消费 topic 从 `nsm-incident-task` → `nsm-aggregated-alert`
  - `handleTask` 兼容 Flink 格式（`incident_id`/`model_id`）和 Node.js 格式（`incidentId`/`modelId`）
  - `kafka.js` 添加 `nsm-aggregated-alert` 到 REQUIRED_TOPICS

- Kafka Topic
  - 新增 `nsm-aggregated-alert`（Flink Aggregator → Node.js Searcher）

- 联调验证
  - ✅ Flink EngineMainEntry 规则匹配（内网 IP → rule-001）
  - ✅ Flink AggregatorFunction 聚合（26 条告警 → 4 个 incident）
  - ✅ ES incident-index + alert-index 写入成功
  - ✅ nsm-aggregated-alert 输出含 incident_id + model_id
  - ✅ Node.js Searcher 消费聚合告警
