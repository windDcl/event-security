# 安全事件模型与安全事件需求文档 v0.1

## 1. 背景

系统属于“网络安全大数据”平台下的“安全引擎”模块，核心目标是通过安全事件模型将告警、日志、实体、威胁情报等数据聚合为安全事件，并提供安全运营人员研判、查询、图谱分析、证据查看和影响面分析能力。

本期覆盖两个核心模块：

- 安全事件模型
- 安全事件

### 3.2 新建模型

字段：

- 模型名称，必填
- 是否使用入口告警名称替代模型名称作为安全事件标题
- 模型描述
- 攻击场景，必填
- 入口告警，必填
- 模型规则，必填
- 激活：是 / 否
- 通知对象

入口告警：

- 默认入口为 A
- 支持添加多个入口告警
- 每个入口告警支持 HQL 或规则表达式
- 支持删除入口告警

模型规则使用 DSL / YAML 结构。

示例结构：

```yaml
processList:
  - condition: ...
    source: "告警"
    time: ["-10m", "10m"]
    relationship: "相同的主机和进程"
    processUnit:
      processList:
        - condition: ...
          source: "日志"
          time: ["-1h", "1h"]
```

规则能力：

- 支持 `processList`
- 支持嵌套 `processUnit`
- 支持 `condition`
- 支持 `source`
- 支持 `time`
- 支持 `relationship`
- 支持 `essential`
- 支持变量引用，例如 `${主机IP}`、`${进程pid}`

### 3.3 编辑模型

## 5. 创建安全事件

### 5.1 流程

创建安全事件分两步：

1. 选择日志 / 事件
2. 填写事件详情并提交

### 5.2 第一步：选择日志 / 事件

页面为日志查询弹窗或抽屉。

查询条件：

- 查询类型：日志查询
- 时间范围
- HQL 查询框
- 日志存储集群
- 过滤条件
- 搜索
- 清空

字段模板：

- 日志检索模板
- 已选字段
- 保存为自定义模板

可选字段示例：

- 发生时间
- 事件名称
- 事件级别
- 组织机构
- 源地址
- 目的地址
- 源端口
- 目的端口
- 数据源
- 原始日志

结果表字段：

- 勾选框
- 发生时间
- 事件名称
- 事件级别
- 组织机构
- 源地址
- 目的地址
- 操作

规则：

- 用户至少选择 1 条日志或事件后，才能进入下一步。
- 单条记录支持查看详情。
- 支持分页。

### 5.3 第二步：填写事件详情

字段：

- 事件名称，必填
- 攻击场景，必填
- 责任人
- 严重等级
- 解决办法

操作：

- 返回
- 确认 / 提交

创建成功后：

- 生成安全事件
- 所选日志 / 事件作为关联数据
- 进入安全事件列表或详情页，待确认

## 7. 自动聚合逻辑

安全事件可由模型自动生成。

流程：

1. 接收合并告警处理任务
2. 判断是新建还是更新
3. 新建时，判断是否匹配已有安全事件
4. 匹配则更新安全事件并合入
5. 不匹配则判断是否命中模型入口告警
6. 命中则创建安全事件
7. 创建后启动异步搜索
8. 更新时，判断是否已创建过某模型的安全事件
9. 已创建则更新安全事件字段
10. 未创建则回到入口告警判断逻辑
