# 安全事件模块拆分与开发计划 v0.1

## 1. 模块划分原则

本系统建议按“业务边界 + 数据依赖 + 可并行开发”拆分模块，而不是单纯按页面拆分。

核心原则：

- 安全事件模型和安全事件分开开发，但通过 `ice_rule_id` 建立关联。
- 列表、创建、详情拆分，降低单模块复杂度。
- HQL 查询、字段模板、表格字段配置作为通用能力沉淀。
- 威胁图谱、证据、影响面作为详情页的独立子模块，后续可渐进增强。
- 自动聚合引擎可以先用接口/任务占位，后续接入 Flink 或异步任务。

## 2. 推荐开发模块

### M1. 基础框架与权限模块

职责：

- 搭建后台页面框架
- 接入顶部导航和左侧菜单
- 接入登录态/用户信息
- 定义权限点
- 统一接口请求、错误处理、Loading、空状态

涉及页面：

- 安全引擎主框架
- 事件聚合分析引擎菜单

主要权限点：

- 查看安全事件
- 创建安全事件
- 编辑安全事件
- 查看安全事件模型
- 新建/编辑模型
- 启停模型
- 删除模型
- 导入/导出模型
- 查看/下载证据

优先级：P0

依赖：无

交付标准：

- 页面框架可访问
- 菜单结构完整
- 权限点可配置或可 Mock

## 5. 创建安全事件域

### M9. 创建安全事件模块

职责：

- 从安全事件列表进入创建流程
- 第一步选择日志/事件
- 第二步填写事件详情
- 提交创建安全事件
- 将所选日志/事件绑定为关联数据

流程：

1. 打开日志查询弹窗
2. 查询日志或事件
3. 勾选至少一条记录
4. 下一步
5. 填写事件详情
6. 提交创建

表单字段：

- 事件名称，必填
- 攻击场景，必填
- 责任人
- 严重等级
- 解决办法

后端接口：

- `POST /api/security/incidents`
- `POST /api/security/logs/search`

优先级：P0

依赖：M2、M6、M7

交付标准：

- 未选择日志/事件时不能进入下一步
- 必填项校验可用
- 创建后写入安全事件
- 创建后事件可在列表查询到

## 7. 自动聚合与规则执行域

### M16. DSL 校验与编译模块

职责：

- 校验模型 DSL 语法
- 校验变量引用
- 校验 `processList` / `processUnit` 结构
- 校验 `source`、`time`、`relationship`
- 生成可执行 `statement`

后端接口：

- `POST /api/security/models/validate`

优先级：P1

依赖：M4

交付标准：

- 保存模型前可校验 DSL
- 错误可定位到字段或行

### M17. 自动聚合任务模块

职责：

- 判断入口告警是否命中模型
- 判断新建或更新事件
- 创建安全事件
- 更新安全事件
- 启动异步关联搜索

数据输入：

- Kafka 告警/日志流
- ES 告警/日志索引
- PostgreSQL 模型配置

数据输出：

- 安全事件索引
- 关联告警/日志关系
- 图谱节点边
- 分析任务进度

优先级：P2

依赖：M3、M4、M16

交付标准：

- 可根据入口告警创建事件
- 可合入已有事件
- 可记录任务状态

### M18. 异步关联搜索模块

职责：

- 执行模型 DSL 中的 `processList`
- 搜索关联告警、日志、实体、证据
- 生成图谱关系
- 更新安全事件详情数据

后端/任务接口：

- `POST /api/security/incidents/{id}/analysis-tasks`
- `GET /api/security/incidents/{id}/analysis-tasks/{taskId}`

优先级：P2

依赖：M16、M17

交付标准：

- 可按模型规则异步扩展事件上下文
- 可查看任务进度

## 11. 并行开发建议

可并行小组：

- 前端 A：M3、M4、M5，负责安全事件模型域
- 前端 B：M7、M8、M9，负责安全事件列表和创建
- 前端 C：M10、M11、M12、M13、M14、M15，负责事件详情
- 后端 A：M2、M3、M4、M5，负责模型和字典
- 后端 B：M6、M7、M8、M9，负责事件查询和创建
- 后端 C：M10-M15，负责详情聚合查询
- 引擎/数据：M16、M17、M18，负责 DSL 和自动聚合

接口契约优先级：

1. M2 字典接口
2. M3/M4 模型接口
3. M7 事件列表接口
4. M6 日志查询接口
5. M9 创建事件接口
6. M10-M15 详情接口

## 12. 风险点

- HQL 语法和 ES 查询语法转换需要统一。
- 模型 DSL 的完整语法目前只确认了核心结构，仍需补充校验规范。
- 事件详情数据来自多个索引，接口聚合复杂度较高。
- 威胁图谱若节点过多，需要做默认裁剪和按需展开。
- 自动聚合和人工创建事件需要统一数据结构，否则详情页会分叉。
