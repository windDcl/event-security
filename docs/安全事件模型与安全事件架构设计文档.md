# 安全事件模型与安全事件架构设计文档 v0.1

## 1. 目标与范围

本文档用于指导“安全事件模型”和“安全事件”模块的复刻实现。系统需要支持安全事件模型配置、模型规则解析、自动聚合生成安全事件、人工创建安全事件、事件检索、详情研判、威胁图谱、证据和影响面分析。

本期架构覆盖：

- 安全事件模型管理
- 安全事件列表与检索
- 人工创建安全事件
- 自动聚合生成安全事件
- 安全事件详情五个 Tab
- 模型 DSL 解析与执行
- PostgreSQL、Elasticsearch、Kafka、Redis/Flink 相关集成

## 2. 总体架构

系统采用“管理后台 + 事件聚合服务 + 实时检测引擎 + 检索存储”的分层架构。

## 3. 核心模块

### 3.1 Web 前端

前端负责后台操作界面，包含以下页面：

- 安全事件模型列表
- 安全事件模型新建/编辑
- 安全事件模型详情
- 安全事件列表
- 创建安全事件两步流程
- 安全事件详情
- HQL/日志查询弹窗
- 表格字段配置弹窗

前端不直接访问数据库或 Elasticsearch，所有数据通过后端 API 获取。

### 3.2 安全事件后端服务

后端服务是前端与存储/引擎之间的统一访问层，职责包括：

- 模型 CRUD
- 攻击场景查询
- 模型启停、导入、导出
- 模型 DSL 校验与保存
- 安全事件列表查询
- 安全事件详情聚合查询
- 人工创建安全事件
- HQL 日志查询
- 图谱、证据、影响面查询
- 调用或触发异步关联搜索任务

### 3.3 事件聚合与检测引擎

事件聚合与检测引擎负责自动生成安全事件。

现有工程中已有 `nsm-event-engine`，其核心能力包括：

- 从 Kafka 消费安全日志/告警
- 通过规则表达式树匹配 HQL 条件
- 使用 Flink Side Output 分发多种规则类型
- 将匹配结果输出到 Kafka
- 定时从 PostgreSQL 加载规则配置
- 使用 Redis 查询白名单和富化信息

复刻时建议保留该设计思路，将“安全事件模型”作为更高层的事件聚合规则。

## 5. 模型 DSL 架构

### 5.1 DSL 结构

模型 DSL 使用 YAML/JSON 风格结构。

```yaml
processList:
  - condition: 主机IP = "${主机IP}" and 进程pid = "${进程pid}"
    source: "告警"
    time: ["-10m", "10m"]
    relationship: "相同的主机和进程"
    essential: true
    processUnit:
      processList:
        - condition: 源地址 = "${主机IP}" or 目的地址 = "${主机IP}"
          source: "日志"
          time: ["-1h", "1h"]
          relationship: "源地址或目的地址关联到主机IP"
```

### 5.2 DSL 字段说明

- `processList`：关联搜索步骤列表
- `condition`：HQL 风格条件表达式
- `source`：查询来源，如“告警”“日志”
- `time`：相对入口告警的时间窗口
- `relationship`：关系描述，展示在告警关联条件和图谱边上
- `essential`：是否核心过程
- `processUnit`：嵌套流程，用于继续扩展关联搜索

### 5.3 DSL 执行过程

1. 入口告警命中模型入口条件。
2. 以入口告警字段作为变量上下文。
3. 解析 `processList`。
4. 将 `${变量}` 替换为入口告警或上一步结果中的字段值。
5. 按 `source` 查询告警或日志索引。
6. 按 `time` 限制查询时间窗口。
7. 将查询结果写入安全事件关联数据。
8. 若存在 `processUnit`，递归执行下一层关联搜索。
9. 生成图谱节点、边和关系描述。

## 6. 核心业务流程

### 6.1 自动聚合生成安全事件

```text
合并告警进入处理流程
        │
        ▼
判断是新建还是更新
        │
        ├─ 新建
        │   ├─ 是否匹配已有安全事件
        │   │   ├─ 是：更新安全事件并合入
        │   │   └─ 否：判断是否命中模型入口告警
        │   │       ├─ 是：创建安全事件并启动异步搜索
        │   │       └─ 否：跳过处理
        │
        └─ 更新
            ├─ 是否创建过某模型的安全事件
            │   ├─ 是：更新安全事件字段
            │   └─ 否：判断是否命中模型入口告警
```

### 6.2 人工创建安全事件

```text
安全事件列表
   │
   ├─ 点击“创建安全事件”
   │
   ▼
日志查询弹窗
   │
   ├─ 输入 HQL / 选择时间范围 / 选择字段
   ├─ 查询日志或事件
   ├─ 勾选一条或多条记录
   │
   ▼
填写事件详情
   │
   ├─ 事件名称
   ├─ 攻击场景
   ├─ 责任人
   ├─ 严重等级
   ├─ 解决办法
   │
   ▼
提交创建
   │
   ▼
写入安全事件索引，并建立关联日志/告警关系
```

### 6.3 安全事件详情聚合查询

安全事件详情页不是单索引直出，而是聚合多个数据源：

- 事件基础信息：安全事件索引
- 告警 Tab：关联告警/日志索引
- 威胁图谱：事件实体、告警关系、DSL relationship
- 相关证据：威胁情报、恶意样本、可疑文件
- 影响面：主机、内网 IP、账号等实体索引

## 10. 非功能设计

### 10.1 性能

- 安全事件列表默认分页，避免深分页；大页码建议使用 search_after。
- 详情页 Tab 懒加载。
- 威胁图谱限制默认节点数量，支持展开更多。
- HQL 查询限制默认时间范围，避免无条件大范围检索。

### 10.2 可靠性

- Flink 检测引擎使用 checkpoint。
- 自动聚合任务需要幂等，避免重复创建同一安全事件。
- 人工创建安全事件提交需防重复点击。
- 模型启停和规则加载需要版本号，避免旧规则覆盖新规则。

### 10.3 安全

- HQL 查询必须做字段白名单和语法校验。
- DSL 保存前必须校验结构、变量和时间窗口。
- 下载证据需要权限控制和审计。
- 日志详情、原始日志应按敏感字段脱敏策略展示。
